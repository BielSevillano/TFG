*   **Preservation of Original Input:** Using `n2 = n` is good practice as it allows the final output to display the original number, even though `n` itself is modified within the loop.

**Weaknesses:**
*   **Duplicate Output for n=0 (Reason for "Wrong Answer"):** The primary reason for the "Wrong Answer" verdict is likely a redundant `print` statement. When `n` is `0`, the line `print('El nombre de digits de', n, 'es', d)` inside the `if n == 0:` block is executed. Subsequently, the line `print('El nombre de digits de', n2, 'es', d)` *outside* the `if-else` block is also executed, leading to two identical lines of output for `n=0`. Competitive programming judges typically expect exactly one output line in such cases. For `n > 0`, only the final `print` statement is executed, which is correct.

**3. Suggestions for Improvement**

1.  **Consolidate Print Statements (Recommended Fix):** The most direct fix is to ensure the output is printed only once. This can be achieved by removing the `print` statement from inside the `if` block, allowing the final `print` statement to cover all cases correctly.

    ```python

    n2 = n # Stores the original value of n for printing later

    ```

### Suggestions for Improvement

While the current solution is perfectly acceptable and likely the most idiomatic Python for this problem, here are a few alternative approaches that also solve the problem and might be worth considering for learning purposes or slightly different constraints:

1.  **Explicit Conversion to Integer and then String:**
    ```python
    n_str = input()
    n_int = int(n_str)
    print("The number of digits of", n_int, "is", str(len(n_str)) + ".")
    ```
    This is functionally identical to the provided solution but explicitly shows the conversion from input string to integer, which might be more robust if subsequent operations on the number were needed. However, for *just* counting digits, it's redundant.

2.  **Mathematical Approach (Iterative):**
    ```python
    n_str = input() # Read as string to preserve original for printing
    n = int(n_str)
    count = 0
    if n == 0:
        count = 1
    else:
        while n > 0:
            n //= 10  # Integer division by 10
            count += 1
    print("The number of digits of", n_str, "is", str(count) + ".")
    ```
    This approach uses integer division in a loop. It's a common method in languages that don't have a direct string length equivalent for numbers. It correctly handles the case of `n=0`. This is a good exercise to understand number manipulation.

3.  **Mathematical Approach (Logarithmic):**
    ```python
    import math

    n_str = input() # Read as string to preserve original for printing
    n = int(n_str)

    if n == 0:
        count = 1
    else:
        count = int(math.log10(n)) + 1

    print("The number of digits of", n_str, "is", str(count) + ".")
    ```
    This is a more advanced mathematical solution using logarithms. `log10(n)` gives the power to which 10 must be raised to get `n`. For example, `log10(100)` is 2, and `log10(999)` is approximately 2.99. Taking the integer part and adding 1 gives the number of digits. This is very efficient but requires understanding logarithms and handling the `n=0` edge case separately.
